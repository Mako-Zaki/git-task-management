# タスク管理システム使い方ガイド

## クイックスタート

### 毎日の使い方

1. **朝: タスクの確認**
   ```bash
   cd ~/git-task-management
   git pull origin main
   open TODO.md  # または お好きなエディタで開く
   ```

2. **タスク追加時**
   - TODO.mdを編集
   - 適切なカテゴリに追加
   - 期限がある場合は記載

3. **変更を保存**
   ```bash
   git add TODO.md
   git commit -m "タスク追加: 〇〇"
   git push origin main
   ```

4. **完了時**
   - チェックボックスをチェック
   - 完了セクションに移動
   - 完了日を記載
   - コミット＆プッシュ

## タスクの書き方のベストプラクティス

### ✅ 良い例

```markdown
- [ ] 機械学習の論文3本読む (期限: 2026-02-20)
- [ ] リクルート社ES提出「学生時代頑張ったこと」400字 (締切: 2026-02-25)
- [ ] 歯医者予約 (希望: 今週中)
```

### ❌ 悪い例

```markdown
- [ ] 論文読む（どの論文？何本？）
- [ ] ES書く（どこの会社？締切は？）
- [ ] 病院（何科？いつまでに？）
```

### ポイント

- **具体的に** - 何をするか明確に
- **期限を記載** - いつまでにやるか
- **サイズ感** - 大きすぎるタスクは分割
- **行動ベース** - 「〇〇について考える」より「〇〇のメモを書く」

## カテゴリの使い分け

### 🔥 緊急・重要

- 今日中、明日までにやるべきこと
- 締切が3日以内のもの
- 忘れると大きな問題になるもの

### 🎓 研究関連

- 実験、データ分析
- 論文執筆、レビュー
- ゼミ発表準備
- 文献調査

### 💼 就職活動

#### ES・書類提出
- エントリーシート
- 履歴書
- ポートフォリオ更新

#### 面接・説明会
- 企業研究
- 面接対策
- 説明会参加

### 📅 日常・予定

- 買い物リスト
- 役所の手続き
- イベント参加
- 家事

### 💡 個人プロジェクト

- プログラミング学習
- 趣味の制作
- 副業
- スキルアップ

## よくある使い方

### パターン1: 就活のES管理

```markdown
## 💼 就職活動

### ES・書類提出

- [ ] リクルート社ES (締切: 2026-02-25)
  - [ ] 自己PR 400字
  - [ ] 学生時代頑張ったこと 600字
  - [ ] 志望動機 800字
- [ ] サイバーエージェント社ES (締切: 2026-03-01)
```

大きなタスクをサブタスクに分割すると管理しやすい。

### パターン2: 研究の進捗管理

```markdown
## 🎓 研究関連

### 進行中

- [ ] 卒論Chapter 3執筆 (開始: 2026-02-11、目標: 2026-02-28)
  - [x] データ収集
  - [x] データクリーニング
  - [ ] 統計分析
  - [ ] グラフ作成
  - [ ] 考察執筆
```

### パターン3: 週次レビュー

毎週日曜日にやると良いこと：

```bash
# 1. 今週完了したタスクを確認
git log --since="1 week ago" --grep="完了" --oneline

# 2. 来週のタスクを整理
# TODO.mdを開いて、緊急セクションを更新

# 3. コミット
git add TODO.md
git commit -m "週次レビュー: 来週のタスク整理"
git push origin main
```

## スマホでの確認方法

### iOSの場合

1. **Working Copy**アプリを使用（推奨）
   - App Storeからダウンロード
   - GitHubリポジトリをクローン
   - TODO.mdを開いて確認・編集

2. **GitHubアプリ**で確認のみ
   - 編集はできないが閲覧は可能

### Androidの場合

1. **MGit**アプリを使用
2. **GitHubアプリ**で確認

## トラブルシューティング

### コンフリクトが発生した場合

```bash
git pull origin main
# コンフリクトが発生

# TODO.mdを開いて手動でマージ
# <<<<<<< HEAD と >>>>>>> の部分を修正

git add TODO.md
git commit -m "コンフリクト解決"
git push origin main
```

### 間違えてコミットした場合

```bash
# 最後のコミットを取り消す（変更は残る）
git reset --soft HEAD~1

# TODO.mdを修正
git add TODO.md
git commit -m "修正したコミットメッセージ"
git push origin main
```

## おすすめの習慣

1. **毎朝**: 今日のタスクを確認・追加
2. **タスク完了時**: すぐにチェックしてコミット
3. **毎晩**: 明日のタスクを準備
4. **毎週日曜**: 来週の予定を整理
5. **月末**: 完了タスクをアーカイブ

## さらに便利に使うために

### エイリアスの設定

`.zshrc`や`.bashrc`に追加：

```bash
alias todo="cd ~/git-task-management && git pull && open TODO.md"
alias todopush="cd ~/git-task-management && git add TODO.md && git commit -m 'タスク更新' && git push origin main"
```

使い方：

```bash
todo      # TODO.mdを開く
todopush  # 変更をプッシュ
```

### VSCodeで開く場合

```bash
alias todo="cd ~/git-task-management && git pull && code TODO.md"
```
